1. Introduction & Objectives

  • The Problem: Traditional risk assessments rely on static statistical models that fail to capture individual metabolic variability.
  • The Gap: Many healthcare digital twins rely on complex physiological simulations that hinder scalability and real-time clinical use.
  • Our Solution: An interpretable, machine learning-driven Digital Twin (DT) framework using nationally representative data.
  • Deliverable: A reproducible pipeline + API + React dashboard enabling real-time prediction and actionable what‑if analysis.

2. Data & Digital Twin Encoding

The framework constructs patient-level records from NHANES and encodes each patient as a compact “digital twin” state.

  • Data Source: NHANES 2017–2018.
  • Processed cohort: 2,899 participants.
  • Inputs: Age, Gender, BMI, BP, Glucose, Insulin, Total Cholesterol, HDL.
  • Label: Metabolic risk derived from HbA1c thresholding (target creation only).

4. Model Training & Evaluation

The deployed model is a scikit‑learn pipeline (imputation → scaling → logistic regression). Hyperparameters are tuned using 5‑fold stratified CV (objective: F1).
